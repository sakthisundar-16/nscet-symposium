# SYNTAX 2K26 — cPanel Deployment Guide

## ✅ How the Build Works

Running `npm run build` inside the `frontend/` folder outputs directly into the **project root** (`cseworkshop/`):

```
cseworkshop/                ← upload this entire folder to public_html
  ├── index.html            ← React SPA (auto-generated by Vite)
  ├── reactapp/             ← React JS/CSS bundles (auto-generated)
  ├── api/                  ← PHP backend (untouched)
  ├── includes/             ← PHP includes (untouched)
  ├── assets/               ← Your PHP assets (untouched)
  ├── uploads/              ← File upload storage
  ├── .htaccess             ← Handles SPA routing + security
  └── frontend/             ← Source code (exclude from upload)
```

## 🚀 Step-by-Step Deployment

### Step 1 — Build the React app

Open terminal in the `frontend/` folder and run:

```bash
cd cseworkshop/frontend
npm install
npm run build
```

After this completes you'll see `index.html` and `reactapp/` appear in `cseworkshop/`.

### Step 2 — Prepare files for upload

**Include** (upload to cPanel):
- `index.html`
- `reactapp/`
- `api/`
- `includes/`
- `assets/`
- `uploads/` (create empty folder if missing)
- `.htaccess`
- Any PHP files in root (e.g., error pages)

**Exclude** (do NOT upload):
- `frontend/` (source code, not needed on server)
- `frontend/node_modules/`
- `.git/`

### Step 3 — Upload via cPanel File Manager

1. Log in to **cPanel** → **File Manager**
2. Navigate to `public_html/`
3. If deploying to root: upload contents directly into `public_html/`
4. If deploying to subdirectory (e.g., `syntax2k26`): create folder and upload there

### Step 4 — Configure base URL (if in subdirectory)

If your site is at `yourdomain.com/syntax2k26/` (not root), open `frontend/vite.config.ts` and add inside `build`:

```ts
base: '/syntax2k26/',
```

Then rebuild and re-upload.

### Step 5 — Set up the database

1. In cPanel → **MySQL Databases** → create a new database + user
2. Assign all privileges to the user
3. Import your SQL schema via **phpMyAdmin**
4. Update `api/config.php` with your credentials:
   ```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'your_db_name');
   define('DB_USER', 'your_db_user');
   define('DB_PASS', 'your_db_pass');
   ```

### Step 6 — Enable HTTPS (recommended)

In cPanel → **SSL/TLS** → Install a Let's Encrypt certificate.

Then uncomment these lines in `.htaccess`:
```apache
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

### Step 7 — Set folder permissions

In cPanel File Manager, right-click `uploads/` → **Permissions** → set to `755`.

## 🔧 Troubleshooting

| Problem | Fix |
|---|---|
| React routes show 404 | Check `.htaccess` is uploaded and mod_rewrite is enabled |
| API calls fail | Verify `api/config.php` DB credentials |
| Images not loading | Ensure `assets/images/` folder is uploaded |
| Upload fails (large files) | `.htaccess` sets `upload_max_filesize 12M` — contact host if needed |
| White screen | Open browser console — likely a base URL mismatch (Step 4) |
